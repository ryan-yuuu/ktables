1"""KafkaTable — materialize a Kafka topic into an in-memory dict. 

2 

3A generic, self-contained "GlobalKTable at home": every process that runs a 

4table replays the topic from the beginning into a local last-write-wins dict 

5keyed by the Kafka message key, then keeps consuming for live updates. A 

6record with a null value is a tombstone and deletes the key. 

7 

8Verified design facts (traced through aiokafka 0.13.0 source during review): 

9- ``group_id=None`` + constructor topic: assignment is populated synchronously 

10 inside ``consumer.start()`` (NoGroupCoordinator) — no race with our seek. 

11- The catch-up gate (positions vs a start-time ``end_offsets`` snapshot) 

12 survives compaction holes and transaction control markers: the fetcher 

13 advances ``position()`` past compacted-away offsets and control batches. 

14- Partition assignment is NOT fixed at start: NoGroupCoordinator listens for 

15 metadata changes and auto-assigns new partitions (they replay from earliest). 

16 While still catching up, the gate extends itself to newly seen partitions; 

17 after the latch, new partitions' records simply arrive as live updates. 

18- Broker restarts recover transparently (no group/session to lose). A 

19 NON-retriable reader error (e.g. authorization) kills the reader task: that 

20 failure is captured, logged loudly, and surfaced via ``status``/``failure``. 

21 

22Consistency contract (the four guarantees): 

231. After ``start()``/``async with``: complete as of the start-time end offsets 

24 (unless ``status == "degraded"`` — catch-up timed out, view may be partial). 

252. Thereafter: eventually consistent; publish-to-visible is typically a few ms. 

263. Contents are stable between *your* awaits (single event loop; only the 

27 reader task mutates). ``snapshot()`` for copies you hold across awaits. 

284. NO read-your-own-writes: after ``await writer.set(k, v)``, a local 

29 ``table.get(k)`` may return the old value until the broker round trip — 

30 unless you ``await table.barrier()`` first, the on-demand freshness/RYOW 

31 primitive (it proves visibility of every record acked before the call, on 

32 the partitions assigned at call time). 

33 

34See README.md for usage. Integration tests spin up Redpanda automatically via 

35testcontainers (Docker required): ``pytest`` — or ``pytest -m "not integration"`` 

36for the broker-free unit suite. 

37"""

174class KafkaTable(Mapping[str, V]): 

175 """An IMMUTABLE Mapping materialized from a Kafka topic, LWW per key. 

176 

177 Honest dict-likeness: read-only ``Mapping`` only (``table[k]``, ``k in 

178 table``, iteration, ``.get``) — deliberately NOT ``MutableMapping``; writes 

179 go through :class:`KafkaTableWriter`, because there is no 

180 read-your-own-writes (a just-published record is visible only after the 

181 broker round trip). 

182 

183 A running table is a resource handle, not a value: equality is identity 

184 (two tables with momentarily equal contents are not "the same table"). 

185 Not thread-safe; single event loop only. Reads before ``start()`` raise. 

186 """

215 # Optional apply observers for derived views (e.g. GroupedKafkaTable): 

216 # on_set(key, value) fires on an applied value record, on_delete(key) on

226 # its next getmany() (~poll_timeout_ms). Lower BOTH to minimize barrier latency; 

227 # the cost is more frequent fetches/wakeups (broker traffic + reader CPU). The 

228 # default mirrors aiokafka's. 

229 self._fetch_max_wait_ms = fetch_max_wait_ms

231 # Disable where the app lacks topic-create ACLs (see ensure_topic()).

327 async def wait_until_caught_up(self, timeout: float | None = None) -> bool: 

328 """Wait for replay to reach the start-time end offsets; True if reached. 

329 

330 Returns False on timeout OR if the reader has died (check ``status``). 

331 """ 

332 if self._failure is not None: 

333 return False 

334 # Race catch-up against reader death so a death wakes this 

335 # immediately. No shield: cancelling these waiters is harmless. 

336 caught = asyncio.ensure_future(self._caught_up.wait()) 

337 failed = asyncio.ensure_future(self._failed.wait()) 

338 try: 

339 await asyncio.wait({caught, failed}, timeout=timeout, return_when=asyncio.FIRST_COMPLETED) 

340 finally: 

341 for fut in (caught, failed): 

342 fut.cancel() 

343 return self._caught_up.is_set() and self._failure is None

345 async def barrier(self, timeout: float | None = None) -> bool: 

346 """Wait until the table reflects everything published before this call. 

347 

348 Snapshots the topic's end offsets now (over the partitions assigned at 

349 call time) and waits until the reader has consumed AND applied every 

350 record below them. On ``True``, every record whose publish was 

351 broker-acked before this call was invoked — on a call-time-assigned 

352 partition — is visible in the mapping (or counted in ``stats`` as 

353 keyless/decode-skipped). This is the on-demand read-your-own-writes 

354 primitive: ``await writer.set(k, v); await table.barrier(); table[k]`` 

355 is then guaranteed. 

356 

357 ``timeout`` bounds the whole call (the end-offset snapshot plus the 

358 wait); ``None`` waits indefinitely. Returns ``False`` on timeout, 

359 reader death, ``stop()`` racing the wait, or a broker error/timeout 

360 while snapshotting the end offsets — every "couldn't prove it" path is 

361 a ``False``, never an exception. Raises ``RuntimeError`` only on 

362 lifecycle misuse (table never started, or already stopped). 

363 

364 Runtime partition expansion is out of scope: the guarantee covers the 

365 partitions assigned when ``barrier()`` was called. 

366 """ 

367 self._require_started() 

368 if self._consumer is None: 

369 raise RuntimeError("table stopped — barrier() cannot prove freshness on a stopped table") 

370 if self._failure is not None: 

371 return False 

372 consumer = self._consumer 

373 loop = asyncio.get_running_loop() 

374 deadline = None if timeout is None else loop.time() + timeout 

375 tps = sorted(consumer.assignment(), key=lambda tp: tp.partition) 

376 # Snapshot the call-time end offsets ourselves. end_offsets is a pure 

377 # ListOffsets request/response: it touches no fetch buffers, positions, 

378 # or subscription state, so it is safe to call from this coroutine 

379 # while the reader task runs getmany on the same consumer. position(), 

380 # which the reader's catch-up logic mutates state for and which can 

381 # block ~request_timeout_ms, stays reader-loop-only. Bound the snapshot 

382 # by the remaining budget (end_offsets is otherwise capped only by the 

383 # consumer's request_timeout_ms, ~40s, unrelated to our timeout) and 

384 # map a broker failure to False — an unanswerable ListOffsets is an 

385 # environmental condition, same family as timeout/reader-death. 

386 try: 

387 snapshot_budget = None if deadline is None else max(0.0, deadline - loop.time()) 

388 targets: dict[TopicPartition, int] = await asyncio.wait_for(consumer.end_offsets(tps), timeout=snapshot_budget) 

389 except (asyncio.TimeoutError, KafkaError): 

390 logger.warning( 

391 "barrier() could not snapshot end offsets for topic=%s (broker error/timeout); returning False", 

392 self._topic, 

393 ) 

394 return False 

395 fut: asyncio.Future[None] = loop.create_future() 

396 self._barriers.append((targets, fut)) 

397 failed = asyncio.ensure_future(self._failed.wait()) 

398 try: 

399 wait_budget = None if deadline is None else max(0.0, deadline - loop.time()) 

400 await asyncio.wait({fut, failed}, timeout=wait_budget, return_when=asyncio.FIRST_COMPLETED) 

401 finally: 

402 failed.cancel() 

403 if not fut.done(): 

404 fut.cancel() 

405 # Self-prune: drop our own entry however the await exits (resolved, 

406 # timed out, caller-cancelled, reader death). Self-pruning keeps the 

407 # list clean even after a reader death, when the reader can no 

408 # longer sweep. No await in this block — see the reader loop's 

409 # invariant comment. 

410 self._barriers = [(t, f) for t, f in self._barriers if f is not fut] 

411 return fut.done() and not fut.cancelled() and self._failure is None

415 async def start(self) -> None: 

416 """Start the reader and wait (bounded) for catch-up. 

417 

418 Raises on double-start, on a missing topic, and if the reader dies 

419 during catch-up. On catch-up *timeout* the table stays usable and 

420 keeps consuming (``status == "degraded"``, loudly logged) — mirrors a 

421 boot-gate policy of serve-degraded rather than crash-loop. 

422 

423 After a reader-death failure the table counts as started: reads serve 

424 the frozen last-applied state (possibly empty) rather than raising 

425 "not started" — gate on ``status``/``failure`` for liveness decisions. 

426 """ 

427 if self._started: 

428 raise RuntimeError(f"KafkaTable for topic={self._topic!r} already started") 

429 if self._ensure_topic: 

430 await ensure_topic(self._bootstrap_servers, self._topic, topic_configs=self._topic_configs) 

431 # Constructor topic + group_id=None: all partitions are assigned 

432 # synchronously inside start() — no group, no commits, no rebalance. 

433 # (Manual assign()+partitions_for_topic() hits a stale partition 

434 # cache on fresh topics.) 

435 consumer = AIOKafkaConsumer( 

436 self._topic, 

437 bootstrap_servers=self._bootstrap_servers, 

438 group_id=None, 

439 enable_auto_commit=False, 

440 auto_offset_reset="earliest", 

441 fetch_max_wait_ms=self._fetch_max_wait_ms, 

442 ) 

443 await consumer.start() 

444 self._consumer = consumer 

445 try: 

446 tps = sorted(consumer.assignment(), key=lambda tp: tp.partition) 

447 if not tps: 

448 raise RuntimeError( 

449 f"topic {self._topic!r}: no partitions assigned. Topic missing — or, if ensure_topic ran, " 

450 "check the ensure_topic log above for a masked creation error (e.g. ACLs)." 

451 ) 

452 # Keep: auto_offset_reset only covers the no-valid-position 

453 # path; this makes replay-from-zero unconditional. 

454 await consumer.seek_to_beginning(*tps) 

455 # Gate target: HWM at start; later records are live updates. 

456 end_offsets: dict[TopicPartition, int] = await consumer.end_offsets(tps) 

457 except BaseException: 

458 await consumer.stop() 

459 self._consumer = None 

460 raise 

461 self._started = True 

462 task = asyncio.create_task(self._run(consumer, tps, end_offsets, time.perf_counter()), name=f"kafka-table:{self._topic}") 

463 task.add_done_callback(self._on_reader_done) 

464 self._task = task 

465 if not await self.wait_until_caught_up(timeout=self._catchup_timeout): 

466 if self._failure is not None: 

467 # Reader died during boot: fail start() loudly, cleaned up. 

468 failure = self._failure 

469 await self.stop() 

470 raise RuntimeError(f"KafkaTable reader for topic={self._topic!r} died during catch-up") from failure 

471 self._timed_out = True 

472 logger.error( 

473 "table for topic=%s NOT caught up after %.1fs (applied=%d so far); continuing DEGRADED — data may be incomplete", 

474 self._topic, 

475 self._catchup_timeout, 

476 self._live.records_applied, 

477 )

479 def _on_reader_done(self, task: asyncio.Task[None]) -> None: 

480 if task.cancelled(): 

481 return 

482 exc = task.exception() 

483 if exc is not None: 

484 self._failure = exc 

485 self._failed.set() 

486 logger.error( 

487 "KafkaTable reader for topic=%s DIED (%s); table is FROZEN at last applied state (size=%d, status=failed)", 

488 self._topic, 

489 type(exc).__name__, 

490 len(self._data), 

491 exc_info=exc, 

492 )

494 async def stop(self) -> None: 

495 task, self._task = self._task, None 

496 if task is not None: 

497 task.cancel() 

498 try: 

499 await task 

500 except asyncio.CancelledError: 

501 pass 

502 except Exception: 

503 # Logged by _on_reader_done; re-raising here would mask 

504 # caller errors and skip consumer cleanup. 

505 pass 

506 # A cancelled reader task never resolves pending barriers, so wake them 

507 # ourselves: a timeout=None barrier racing shutdown would otherwise hang 

508 # forever. Each woken barrier() returns False and self-prunes; clearing 

509 # here makes the list empty immediately regardless. 

510 for _, fut in self._barriers: 

511 if not fut.done(): 

512 fut.cancel() 

513 self._barriers.clear() 

514 consumer, self._consumer = self._consumer, None 

515 if consumer is not None: 

516 try: 

517 await consumer.stop() 

518 except Exception: 

519 logger.exception("consumer.stop() failed for topic=%s during teardown", self._topic)

530 def _apply(self, record: ConsumerRecord) -> None: 

531 where = f"topic={self._topic} partition={record.partition} offset={record.offset}" 

532 key_bytes = record.key 

533 if key_bytes is None: 

534 self._live.keyless_records += 1 

535 logger.warning("keyless record skipped (%s) — producer is writing unkeyed records to a keyed table topic", where) 

536 return 

537 try: 

538 key = self._key_decoder(key_bytes) 

539 except Exception: 

540 self._live.key_decode_errors += 1 

541 logger.exception("undecodable key skipped (%s)", where) 

542 return 

543 if record.value is None: # null value = tombstone (b"" is NOT a tombstone) 

544 self._data.pop(key, None) 

545 self._live.tombstones_applied += 1 

546 if self._on_delete is not None: 

547 self._on_delete(key) 

548 return 

549 try: 

550 value = self._value_decoder(record.value) 

551 except Exception: 

552 # Poison tolerance: keep the prior value, never kill the reader. 

553 self._live.value_decode_errors += 1 

554 logger.exception("undecodable value skipped (key=%s, %s)", key, where) 

555 return 

556 # value MAY be None (a decoder that maps bytes to None) — that is a real 

557 # value, NOT a tombstone: tombstones are record.value is None (above). 

558 self._data[key] = value 

559 self._live.records_applied += 1 

560 if self._on_set is not None: 

561 self._on_set(key, value)

563 async def _run( 

564 self, 

565 consumer: AIOKafkaConsumer, 

566 tps: list[TopicPartition], 

567 end_offsets: dict[TopicPartition, int], 

568 started: float, 

569 ) -> None: 

570 # Escaping exceptions are captured by _on_reader_done (status 

571 # "failed"); transient outages don't raise — getmany returns empty. 

572 while True: 

573 batches = await consumer.getmany(timeout_ms=self._poll_timeout_ms) 

574 for records in batches.values(): 

575 for record in records: 

576 self._apply(record) 

577 if not self._caught_up.is_set(): 

578 # aiokafka auto-assigns new partitions on metadata change; 

579 # pre-latch, extend the gate — post-latch they arrive as 

580 # live updates. 

581 current = consumer.assignment() 

582 if current and set(tps) != current: 

583 new = sorted(current - set(tps), key=lambda tp: tp.partition) 

584 if new: 

585 end_offsets.update(await consumer.end_offsets(new)) 

586 tps = sorted(current, key=lambda tp: tp.partition) 

587 positions = [await consumer.position(tp) for tp in tps] 

588 if all(pos >= end_offsets[tp] for pos, tp in zip(positions, tps, strict=True)): 

589 self._live.catch_up_seconds = time.perf_counter() - started 

590 self._live.replayed_at_catch_up = self._live.records_applied + self._live.tombstones_applied 

591 self._caught_up.set() 

592 if self._barriers: 

593 # Read positions only for partitions that some pending barrier 

594 # targets, in the same coroutine that runs _apply and after it, 

595 # so "positions reached => records applied" holds by 

596 # construction. Assignment is grow-only under group_id=None 

597 # EXCEPT for transient shrinks: a metadata blip can momentarily 

598 # empty the assignment, making position() raise 

599 # IllegalStateError. Catch it and omit that tp (its barrier 

600 # defers via the .get(tp, -1) default below) rather than letting 

601 # a blip aiokafka self-heals from kill the reader for good. 

602 needed = {tp for targets, _ in self._barriers for tp in targets} 

603 barrier_positions: dict[TopicPartition, int] = {} 

604 for tp in needed: 

605 try: 

606 barrier_positions[tp] = await consumer.position(tp) 

607 except IllegalStateError: 

608 pass # transiently unassigned: defer barriers needing tp 

609 # INVARIANT: the authoritative read is the `for` loop below, and 

610 # there is no await between it and the rebind, so the 

611 # read-modify-write of self._barriers is atomic w.r.t. the event 

612 # loop. The awaits above are the only yield points: a barrier 

613 # appended (or self-pruned) during them is still seen by the for 

614 # loop, and if its target is absent from barrier_positions it 

615 # defers via the .get(tp, -1) default below. 

616 still_pending: list[tuple[dict[TopicPartition, int], asyncio.Future[None]]] = [] 

617 for targets, fut in self._barriers: 

618 if fut.cancelled(): 

619 continue # timed-out / caller-cancelled: drop, never re-add 

620 # .get(tp, -1): a target tp can be absent from 

621 # barrier_positions — a barrier appended during the awaits 

622 # above, or a tp whose position() raised and was skipped; -1 

623 # fails the >= check and defers, never resolving on missing data. 

624 if all(barrier_positions.get(tp, -1) >= off for tp, off in targets.items()): 

625 if not fut.done(): 

626 fut.set_result(None) 

627 else: 

628 still_pending.append((targets, fut)) 

629 self._barriers = still_pending

632class KafkaTableWriter(Generic[V]): 

633 """Writer counterpart of :class:`KafkaTable`: keyed upserts + tombstones. 

634 

635 ``set(key, value)`` publishes the encoded value under the key (the table's 

636 LWW upsert); ``delete(key)`` publishes a null-value tombstone. A periodic 

637 re-``set`` of the same value is a heartbeat — no separate API needed. 

638 

639 Registry-grade durability by default: ``enable_idempotence=True`` (which 

640 implies ``acks=all``), so a leader failover can't drop an acked record and 

641 producer retries can't duplicate or reorder. Opt out for throwaway data. 

642 

643 The key encoder must be deterministic and stable across processes and 

644 versions — on a multi-partition topic, per-key LWW ordering holds only if 

645 a key always hashes to the same partition. 

646 """
